The Phase 4 work itself (see `docs/REMEDIATION_REPORT.md` for the full
write-up) covered four architectural gaps and is reported as complete and
passing:

1. **Memory-coalescing LSU** (`rtl/memory/titan_x5_lsu.v`, new) — replaces a
   broken SM datapath that only ever issued thread 0's address for an entire
   warp. Coalesces up to 32 lane addresses into minimal 128-byte-aligned
   transactions.
2. **IEEE-754 FPU compliance** (`rtl/fpu/titan_x5_fp32_add.v`,
   `rtl/fpu/titan_x5_fp32_mul.v`, rewritten `rtl/tensor/titan_x5_fp16_mul.v`)
   — replaces integer arithmetic on float bit patterns with real NaN/Inf/
   subnormal handling, 4 rounding modes, and exception flags. **Known
   deviation:** FMA is currently `round(round(a*b) + c)` — a double-rounding
   cascade, not a true single-rounding fused multiply-add.
3. **MESI cache coherency** — a real per-SM L1 (`rtl/memory/titan_x5_l1_cache.v`,
   rewritten) plus a new `titan_x5_coherent_xbar` snooping bus (bottom of
   `rtl/interconnect/titan_x5_crossbar.v`) and the L2 instantiated for the
   first time, bridged via the new `rtl/memory/titan_x5_l2_mem_adapter.v`.
   The crossbar **fully serializes** transactions — no split transactions or
   multiple outstanding requests yet (documented as future work).
4. **Bilinear texture filtering** (`rtl/graphics/titan_x5_tmu.v`) — fixed
   raw-blending-before-format-expansion and truncation-instead-of-rounding
   bugs.

All four got dedicated cocotb testbenches under `tb/uvm/` (`test_lsu.py`,
